# Contributing to Contrast Buddy

Thank you for your interest in contributing to Contrast Buddy! We're excited to have you on board.

## Code of Conduct

- Be respectful and inclusive
- Welcome newcomers and help them learn
- Focus on constructive feedback
- Celebrate diversity of ideas and backgrounds

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/yourusername/contrast-buddy.git`
3. Create a branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Commit: `git commit -m 'Add your feature'`
7. Push: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Guidelines

### Code Style

- Use meaningful variable and function names
- Add comments for complex logic
- Follow existing code patterns
- Keep functions focused and small
- Use modern ES6+ features

### Testing

- Test your changes in multiple browsers
- Test on different screen sizes
- Verify accessibility with keyboard navigation
- Check color contrast of your UI changes

### Commit Messages

Use clear, descriptive commit messages:

```
feat: Add color palette export feature
fix: Resolve contrast calculation edge case
docs: Update README with new features
style: Improve button hover animations
refactor: Simplify color validation logic
```

## Pull Request Process

1. Update documentation if needed
2. Ensure code follows project style
3. Test your changes thoroughly
4. Update CHANGELOG.md if applicable
5. Request review from maintainers

## Questions?

Open an issue or start a discussion - we're here to help!
